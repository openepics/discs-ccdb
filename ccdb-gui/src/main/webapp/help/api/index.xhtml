CCDB: Application Programming Interface

Introduction

CCDB (Central Configuration Database) is used by Facility for Rare Isotope Beams (FRIB) to manage configuration of the facility. Data in CCDB can be accessed programmatically through a RESTful Application Programming Interface (API), The CCDB API, which is described in this document.

Version

The CCDB API follows Semantic Versioning for its version numbers. The current API version number is #{msgs.apiversion}.

Base URI

The base URI for Resource Identifiers, on this server, is #{msgs.apiBaseUrl}

Caching

Currently the #{msgs.title} API does not support caching. However, future versions of the API will support caching mechanisms such as Conditional GETs and Expiration Times.

Authentication and Authorization

Note: Authentication mechanisms of CCDB API will be changed in future. So please be prepared to update you application. To use any part of the CCDB API, an application must first authenticate itself. Instead of the standard HTTP Basic, DIGEST, FORM or other authentication mechanisms, CCDB API requires the following HTTP Header to be sent with every request:

DISCS-Authorization: key:passcode

where key and passcode are supplied by the CCDB team to identify your application. Key and passcode must be separated by the ':' character in the header. You must use a key+passcode pair with only one application (and its versions). To use CCDB API with a different application, you must request a new key+passcode pair.

Encryption

#{msgs.title} API forces the use of HTTPS protocol. If HTTP is used, CCDB will redirect it over to HTTPS. Note that the public key of the CCDB development and QA server may not be signed by trusted Certificate Authorities.

CORS

For security reasons, #{msgs.title} API does not support Cross-origin Resource Sharing (CORS).

Error Codes and Responses

The #{msgs.title} API attempts to return appropriate HTTP Status Codes for each request.
